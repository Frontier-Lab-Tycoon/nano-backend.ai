# LoRA / SFT 최소 머털 모델

## SFT란 무엇인가?

Supervised Fine-Tuning (SFT)는 사전 학습된 베이스 모델을 가져와, 더 작고 작업 전용인 데이터셋에서 레이블이 붙은 예제들로 후학습을 진행하는 것을 의미합니다. 모델은 새로운 데이터의 포맷과 의도를 따르는 법을 배우면서, 사전 학습에서 얻은 일반적인 지식은 유지합니다.

## LoRA란 무엇인가?

Low-Rank Adaptation (LoRA)는 파라미터 효율적 파인튀닝 방법입니다. 베이스 모델의 모든 가중치(십억 개의 파라미터)를 업데이트하는 대신, LoRA는 어텐션(attention)과 피드포워드(feed-forward) 층에 작고 학습 가능한 "adapter" 행렬을 주입합니다.

핵심 아이디어:
- 원래 가중치 업데이트: `W' = W + ΔW`
- LoRA는 근사: `ΔW = A × B`를 사용하며, `A`와 `B`는 작습(rank `r`)
- `W`를 얼리고(freeze) `A`와 `B`만 학습합니다

## 왜 MergeOwl에 LoRA인가?

| 전체 파인튀닝 | LoRA |
|----------------|------|
| 모든 파라미터 업데이트 | 약 0.1–1%의 파라미터만 업데이트 |
| 다중 GPU 또는 24GB+ VRAM 필요 | RTX 3090 1장(24GB)에 손쉬움 |
| 수 GB 체크포인트 생성 | 약 10–100MB adapter 생성 |
| 빠른 실험 다루기 어려움 | 빠른 반복, 비교가 쉽습니다 |

## 실제로 중요한 하이퍼파라미터

- `learning_rate`: adapter가 얼마나 공격적으로 업데이트되는지. 일반적인 범위는 1e-4부터 5e-4.
- `num_epochs`: 데이터셋을 몇 번 도는지. 보통 1–5회.
- `lora_r`: adapter 행렬의 rank. 높을수록 더 많은 용량을 지닌 대신 더 많은 메모리를 사용. 일반적: 8, 16, 32, 64.
- `lora_alpha`: 스케일링 계수. 보통 `2 × r`으로 설정.
- `max_seq_length`: 후학습 중 모델이 보는 최장 길이의 시퀀스. 길수록 더 많은 VRAM 사용.
- `micro_batch_size`: 한 번의 forward/backward에 쓸 샘플 수. GPU VRAM에 의해 제한됩니다.

## 머털 모델 체크리스트

1. 베이스 모델은 얼림(adapter만 학습).
2. inference 시 adapter + 베이스 모델을 병합(merge)하면 마치 전체 파인튀닝한 것처럼 동작합니다.
3. 후학습이 실패하면 `r`이나 `learning_rate`를 바꿔서 재시도할 수 있습니다. 베이스 모델을 다운로드할 필요가 없습니다.
4. adapter가 저장과 비교, 다른 adapter와 병합할 수 있는 과제물입니다.
